– handleActionWithIdentifier:forRemoteNotification:completionHandler:

Implementation of this method enables ‘Push Actions’. Forward this call to Appoxee directly and call completionHandler(); if the returned value is NO, since Appoxee will not handle a complete custom action provided by the developer.

- (BOOL)handleActionWithIdentifier:(nullable NSString *)identifier forRemoteNotification:(nonnull NSDictionary *)userInfo completionHandler:(void ( ^ _Nullable ) ( ))completionHandler

Parameters

identifier

The identifier argument passed by the application delegate.
userInfo

The userInfo argument passed by the application delegate.
completionHandler

The completionHandler argument passed by the application delegate.

Return Value

YES, if Appoxee handled the action, else returns NO. If the returned value is NO, you will need to call completionHandler(); after you finish handling the Push action.
